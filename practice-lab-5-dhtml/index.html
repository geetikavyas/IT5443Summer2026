JavaScript DOM Exercise Demos

Exercise Files

  • Exercise 1: Read and Change HTML Node Values
  • Exercise 2: Counter
  • Exercise 3: Button and Textbox

Things Worth Observing

1. HTML elements can be selected using id

In all three files, JavaScript works because each important HTML element has an id. Students should notice that the id in HTML and the name inside getElementById() must match exactly.

2. JavaScript can read HTML content

In Exercise 1, JavaScript reads values from the webpage. It can read the text of the heading, the URL of the link, the text of the link, the image source, the image alt text, and the button text.

3. JavaScript can change HTML content

In Exercise 1, JavaScript changes the existing webpage values. This shows that JavaScript can update the webpage without opening or editing the HTML file again.

4. Events make the page interactive

All three exercises use a button click event. Students should observe that the code inside the click event runs only after the button is clicked. This is what makes the webpage interactive.

5. Textbox values are read using value

In Exercise 3, the user types something in a textbox. To read textbox input, we use value. Students should remember that form input fields usually use value, not innerHTML.

6. Normal HTML text is read using innerHTML

For elements like headings, paragraphs, spans, and buttons, innerHTML can be used to read and change the text inside the element.

7. Numbers from HTML are usually read as text first

In Exercise 2, the number inside the span looks like a number, but JavaScript reads it as text first. That is why the value should be converted to a number before increasing it.

8. JavaScript can change CSS style

In Exercise 3, JavaScript changes the color of the displayed text to red. This shows that JavaScript can change not only the content of a webpage, but also its appearance.

The biggest lesson is that HTML creates the structure, and JavaScript makes the page respond to the user.
